因为没有学过volatile，于是……

在代码1后两条语句中，i 没有被用作左值。这时候编译器认为 i 的值没有发生改变，所以在（1）语句时从内存中取出 i 的值赋给 j 之后，这个值并没有被丢掉，而是在（2）语句时继续用这个值给 k 赋值。编译器不会生成出汇编代码重新从内存里取 i 的值，这样提高了效率。但要注意：（1）、（2）语句之间 i 没有被用作左值才行。
volatile 关键字告诉编译器 i 是随时可能发生变化的，每次使用它的时候必须从内存中取出 i，的值，因而编译器生成的汇编代码会重新从 i 的地址处读取数据放在 k 中。

volatile表示一个变量也许会被后台程序改变

关键字 volatile 是与 const 绝对对立的。它指示一个变量也许会被某种方式修改，这种方式按照正常程序流程分析是无法预知的（例如，一个变量也许会被一个中断服务程序所修改）。

这个关键字使用下列语法定义：

volatile data-definition;
变量如果加了 volatile 修饰，则会*从内存重新装载内容，而不是直接从寄存器拷贝内容。 *

volatile应用比较多的场合，在中断服务程序和cpu相关寄存器的定义。

假如此时&a=0xFFFF5700，那么&a+1是多少呢？答案是：取决于&a的类型** **

  1. 如果&a是一个指向char型的指针，那么&a+1 = 0xFFFF5701

  2. 如果&a是一个指向int型的指针，那么&a+1 = 0xFFFF5704 (32位)

  3. 如果&a是一个指向某种结构体struct foo的指针，那么&a+1 = 0xFFFF5700+sizeof(struct foo) &a+1 –>** &a+1表示的是指针加法运算**

    int * ptr1 = (int * )(&a + 1) –> ptrl1指向数组a最后一个元素之后的下一个内存单元开始的4个字节（int*)内存

    %x，prt1[-1] –> ptr1[-1]相当于把ptr1往后挪一个整型大小（4字节），即*ptr1[-1]指向数组a的最后一个元素 *–>结果为5

    （2）int ptr2 = (int)((int)a + 1)

    (int)a –> 把数组名a强制转换成整型变量

    (int)a + 1 –> 把整型变量a + 1

    int ptr2 = (int)((int)a + 1) –> ptr2指向a[0]的第二个字节

    %x，*prt2 –> 输出的内容就是ptr2所指向的往后4个字节的内容

    但是机器究竟会把这个数输出成大端字节序呢，还是解释成小端字节序呢？

    答案是：x86平台是小端序的，存取时*最低位 *对应 *低地址 *，因此将会打印出0200 0000

官方解释：

1、所谓大端(big-endian)序，就是高优先位 对应高有效位 。

2、所谓小端(little-endian)序，就是高优先位 对应*低有效位 *。

民间解释：

1、所谓大端(big-endian)序，就是读取或者存放数据时，*最低 位 *对应 *高地址 *。

关于getchar()：
返回值：getchar函数的返回值是用户输入的第一个字符的ASCII码，当出错时返回-1。并且将用户输入的字符回显到屏幕。
